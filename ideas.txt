1) добавлять объекты быстро без "проверки"
 1.1) хранить хэш сырых данных, хэш обработанных данных и хэш конфигурации при которой эти данные были обработаны
 1.2) базовые индексы на коллекцию
    -TcObjectHash уникальный хэш на основе индексируемых данных, предотвращающий дублирование документов
    -TcConfigHash хэш указывающий относительно какой конфигурации документ был обработан
2) демоном обрабатывать объекты
 2.1) на основе сырых данных формировать индексы у новых объектов и объектов, у которых хэш конфигурации отличается от текущего
 2.2) после кажой работы создавать хэш состояния БД для валидации актуальности данных в кэше (или удалять кэш)
3) выплолнять стандартные запросы
 3.1) сбор уникальных значений и количества объектов указанного критерия с указанным условием
 3.2) поиск объектов по указанному условию с сортировкой по указанному критерию
4) кэширование результатов стандартных запросов до момента изменения хэша состояния базы

*) демоном проверять существование в исходной базе объекты из собственной БД с даттой создания больше N
